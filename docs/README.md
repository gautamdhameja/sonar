# Sonar Documentation

Sonar helps you create local-first, source-grounded codebase briefings and ask orientation questions. It is designed for product, business, support, leadership, and semi-technical readers who need project context before talking to engineering.

Sonar is not intended to replace a coding agent, debugger, or deep implementation review. Use it for repository orientation, workflows, risks, source landmarks, and questions to ask next.

Use these chapters:

- [Getting Started](getting-started.md) for installation, first launch, and model setup.
- [Desktop App](desktop.md) for the app flow, repository import behavior, and model settings.
- [Language Support and Limits](language-support.md) for supported parser coverage and what happens with unsupported repositories.
